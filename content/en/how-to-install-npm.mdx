Installing npm is essential for managing JavaScript packages and is typically bundled with Node.js. This guide will walk you through the process of getting npm set up on your system, whether you're a beginner or an experienced developer. For a more detailed walkthrough, see our [complete guide on installing npm](/en/how-to-install-npm).

## What You Need

- Computer (Windows, macOS, or Linux)
- Internet connection
- Administrator privileges
- Terminal or Command Prompt

## Steps

### Step 1: 1. Check for Existing Node.js and npm

Before starting a fresh installation, it's good practice to check if Node.js and npm are already present on your system. Open your terminal or command prompt and execute `node -v` and then `npm -v`. If version numbers are displayed, you might already have them installed, or you may need to update to a newer version.

> 💡 **Tip:** If you find older versions, consider updating them to ensure compatibility with modern JavaScript packages and tools.

### Step 2: 2. Download the Node.js Installer

npm is distributed with Node.js, so your primary step is to install Node.js. Navigate to the official Node.js website (nodejs.org) using your web browser. Download the recommended LTS (Long Term Support) version that is appropriate for your operating system (Windows, macOS, or Linux). The LTS version is stable and generally preferred for most users.

> 💡 **Tip:** Always download installers directly from the official Node.js website to ensure authenticity and security.

### Step 3: 3. Run the Node.js Installer

Once the download is complete, locate the installer file (e.g., .msi for Windows, .pkg for macOS) and run it. Follow the step-by-step instructions provided by the installer. Accept the license agreement and, unless you have specific reasons, choose the default installation options. The installer will automatically set up both Node.js and npm on your system; ensure the 'npm package manager' component is selected.

> 💡 **Tip:** On Windows, the installer automatically adds Node.js and npm to your system's PATH environment variable, making them accessible from any directory in the command prompt.

### Step 4: 4. Verify the Installation

After the installation concludes, it's crucial to verify that both Node.js and npm have been installed correctly. Close and then reopen your terminal or command prompt to refresh the environment variables. Then, type `node -v` and `npm -v` again. You should now see the version numbers for Node.js and npm, confirming a successful installation.

> 💡 **Tip:** If you don't see the version numbers, try restarting your computer or manually checking your system's PATH variable to ensure Node.js and npm executables are included.

### Step 5: 5. Update npm to the Latest Version (Recommended)

While npm comes bundled with Node.js, it often receives updates more frequently than Node.js itself. To benefit from the latest features, bug fixes, and security enhancements, it's a good practice to update npm separately. In your terminal, run the command `npm install -g npm@latest` to upgrade npm globally on your system.

> 💡 **Tip:** The `-g` flag ensures npm is installed globally, allowing you to use it from any project directory on your machine.

### Step 6: 6. Test npm with a Sample Project

To ensure npm is fully functional and ready for use, you can perform a quick test. Create a new, empty directory on your computer, navigate into it using your terminal, and run `npm init -y` to initialize a new npm project. Then, install a simple package like `lodash` by executing `npm install lodash`. This process should create a `node_modules` folder and a `package-lock.json` file, indicating npm is working correctly.

> 💡 **Tip:** This step is a great way for [developers](/en/how-to-install-npm-step-by-step-guide) to ensure their setup is ready for project work and dependency management.


## Tips & Tricks

- Always use the LTS (Long Term Support) version of Node.js for production environments due to its stability and extended support period.
- If you encounter permission errors on macOS or Linux during global npm package installations, you might need to use `sudo` (e.g., `sudo npm install -g <package>`), but use it cautiously.
- Consider using a Node Version Manager (NVM) like `nvm` for macOS/Linux or `nvm-windows` to easily switch between different Node.js and npm versions for various projects. If you're new to the command line, check out our [step-by-step guide for beginners](/en/how-to-install-npm-guide) for more basic instructions.

## Warnings

- ⚠️ Be cautious when running `npm install -g` commands, especially with `sudo`, as global installations can affect system-wide packages and potentially lead to conflicts.
- ⚠️ Only download Node.js installers from the official Node.js website (nodejs.org) to prevent downloading malicious or tampered software.

## Frequently Asked Questions

**How do I install npm?**

You install npm by first installing Node.js, as npm is bundled with it. Download the Node.js installer from the official website, run it, and follow the on-screen instructions to complete the installation of both Node.js and npm.

**How to install npm in Windows?**

To install npm in Windows, download the Windows installer (.msi file) from the official Node.js website. Run the installer, accept the default settings, and ensure the 'npm package manager' component is selected during the installation process.

**How to install npm on Mac?**

For macOS, download the `.pkg` installer from nodejs.org and follow the guided installation steps. Alternatively, you can use Homebrew by running `brew install node` in your terminal, which will install both Node.js and npm.

**How to install npm in Ubuntu?**

To install npm in Ubuntu, it's recommended to use the official NodeSource repositories for an up-to-date version. First, add the repository using curl, then update your package list with `sudo apt update`, and finally install Node.js (which includes npm) using `sudo apt install nodejs`.

**What is npm and why do I need it?**

npm (Node Package Manager) is the default package manager for Node.js and the largest software registry globally. You need it to easily install, manage, and share reusable code packages (dependencies) for your JavaScript projects, streamlining development workflows.

**Can I install npm without Node.js?**

No, you cannot install npm independently without Node.js. npm is tightly integrated and distributed as part of the Node.js installation package. When you install Node.js, npm is automatically included and set up on your system.

**How do I check if npm is installed correctly?**

To verify npm installation, open your terminal or command prompt and type `npm -v`. If a version number is displayed (e.g., 8.x.x), then npm is successfully installed and accessible on your system. If you get an error, it indicates an issue with the installation or your PATH.
