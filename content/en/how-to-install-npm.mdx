Installing npm (Node Package Manager) is typically done by installing Node.js, as npm comes bundled with it. This process involves downloading the appropriate Node.js installer for your operating system and running it. Once Node.js is installed, npm will be available for managing your JavaScript packages and project dependencies.

## What You Need

- Computer with internet access
- Administrator privileges

## Steps

### Step 1: Check for Existing Node.js and npm

Before proceeding, it's a good idea to check if Node.js and npm are already installed on your system. Open your terminal (or Command Prompt on Windows) and type `node -v` then press Enter. Do the same for `npm -v`. If you see version numbers, they are installed. If not, or if the versions are outdated, you'll proceed with the installation.

> 💡 **Tip:** If you have an older version, consider uninstalling it first to avoid conflicts, though the installer usually handles upgrades smoothly.

### Step 2: Download the Node.js Installer

npm is distributed with Node.js, so you need to install Node.js to get npm. Visit the official Node.js website (nodejs.org). You'll typically see two recommended versions: the LTS (Long Term Support) version and the Current version. For most users, the LTS version is recommended for its stability. Download the installer appropriate for your operating system (Windows, macOS, or Linux).

> 💡 **Tip:** Always download from the official Node.js website to ensure you get a secure and legitimate installer.

### Step 3: Install Node.js on Your System

Locate the downloaded installer file and run it. For Windows and macOS, this is usually a `.msi` or `.pkg` file. Follow the prompts in the installation wizard, accepting the default settings for most users. Ensure that the "npm package manager" component is selected for installation, which it usually is by default. Complete the installation process.

> 💡 **Tip:** On Linux, you might use your system's package manager (like `apt` or `yum`) for installation, or follow specific instructions for your distribution. Remember that [How to Install npm: A Step-by-Step Guide for Beginners](/en/how-to-install-npm-guide) provides more detailed OS-specific instructions.

### Step 4: Verify the Installation

After the installation is complete, close and reopen your terminal or Command Prompt. This ensures that your system's PATH variable is updated to include Node.js and npm. Type `node -v` and `npm -v` again. You should now see the version numbers for both Node.js and npm, confirming a successful installation.

> 💡 **Tip:** If you don't see version numbers, try restarting your computer or checking your system's PATH environment variable.

### Step 5: Update npm to the Latest Version

While npm comes bundled with Node.js, it's often updated more frequently. It's a good practice to update npm to its latest stable version after installing Node.js. Open your terminal and run the command: `npm install -g npm@latest`. The `-g` flag ensures npm is installed globally, making it available system-wide.

> 💡 **Tip:** Regularly updating npm helps you access the latest features, bug fixes, and security patches.


## Tips & Tricks

- Always use the LTS (Long Term Support) version of Node.js for production environments due to its stability and long-term maintenance.
- If `npm` commands aren't recognized after installation, try restarting your terminal or even your computer to refresh system paths.
- Consider using a Node Version Manager (like `nvm` for Mac/Linux or `nvm-windows` for Windows) if you need to switch between different Node.js and npm versions for various projects. For more advanced installation scenarios, refer to [How to Install npm: A Step-by-Step Guide for Developers](/en/how-to-install-npm-for-developers).

## Warnings

- ⚠️ Always download Node.js installers from the official nodejs.org website to prevent malware or corrupted files.
- ⚠️ Be cautious when using `sudo` with `npm install` on Linux, as incorrect permissions can cause issues. It's often better to configure npm to work without `sudo` or use a version manager.

## Frequently Asked Questions

**How do I install npm in Windows?**

To install npm in Windows, download the Node.js `.msi` installer from the official Node.js website. Run the installer and follow the prompts, accepting the default options. npm is automatically included with the Node.js installation.

**How do I install npm on Mac?**

On Mac, you can install npm by downloading the Node.js `.pkg` installer from nodejs.org and running it. Alternatively, many developers use Homebrew: `brew install node`. Both methods will include npm with Node.js.

**How do I install npm on Ubuntu?**

To install npm on Ubuntu, it's recommended to use the NodeSource PPA for the latest stable versions. First, update your package list (`sudo apt update`), then add the NodeSource PPA, and finally install Node.js and npm using `sudo apt install nodejs`.

**What is npm and why do I need it?**

npm stands for Node Package Manager. It is the default package manager for Node.js and the world's largest software registry. You need npm to install, manage, and share JavaScript packages (libraries, frameworks, tools) for your projects, making development faster and more efficient.

**Can I install npm without Node.js?**

No, npm is bundled with Node.js and cannot be installed independently as a standalone package for general use. While there are advanced methods for specific scenarios, for typical development, installing Node.js is the standard way to get npm.

**What should I do if npm commands are not recognized after installation?**

First, try closing and reopening your terminal or Command Prompt to refresh your system's PATH. If that doesn't work, restart your computer. On some systems, you might need to manually add the Node.js installation directory to your system's PATH environment variable.
