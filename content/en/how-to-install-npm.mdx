Installing npm is essential for managing JavaScript packages and dependencies in your projects. It typically comes bundled with Node.js, making the installation process straightforward across different operating systems. This guide will walk you through the necessary steps, ensuring you have npm ready for your development projects. For a more detailed walkthrough, see our [How to Install npm: A Step-by-Step Guide](/en/how-to-install-npm).

## What You Need

- A computer (Windows, macOS, or Linux)
- Internet connection
- Terminal or Command Prompt access

## Steps

### Step 1: Check for Existing Node.js and npm

Before starting the installation, it's good practice to check if Node.js and npm are already present on your system. Open your terminal or command prompt and type `node -v` to check the Node.js version, then `npm -v` for the npm version. If version numbers are displayed, they are installed; otherwise, you'll see an error message.

> 💡 **Tip:** If you have an older version of Node.js or npm, you might consider updating or reinstalling for the latest features and security updates.

### Step 2: Download the Node.js Installer

The easiest and recommended way to install npm is by installing Node.js, as npm is automatically bundled with it. Visit the official Node.js website (nodejs.org) and navigate to the downloads section. Download the recommended LTS (Long Term Support) version installer that matches your operating system (Windows, macOS, or Linux).

> 💡 **Tip:** Always download from the official Node.js website to ensure you get a legitimate and secure installer.

### Step 3: Run the Node.js Installation Wizard

Locate the downloaded Node.js installer file and double-click to run it. Follow the prompts in the installation wizard, accepting the license agreement and default installation settings. The installer will automatically configure both Node.js and npm on your system, adding them to your system's PATH.

> 💡 **Tip:** Ensure that the 'npm package manager' component is selected during the installation, though it is typically selected by default.

### Step 4: Verify Node.js and npm Installation

After the installation process is complete, it's crucial to verify that both Node.js and npm have been installed correctly. Close any existing terminal or command prompt windows and open a new one. Type `node -v` and `npm -v` again. You should now see the installed version numbers, confirming a successful setup.

> 💡 **Tip:** If you encounter 'command not found' errors, try restarting your computer to ensure environment variables are fully updated.

### Step 5: Update npm to the Latest Version (Recommended)

While npm comes bundled with Node.js, it is often updated more frequently. To ensure you're using the latest features, bug fixes, and security patches, it's a good practice to update npm itself. Open your terminal and run the command `npm install -g npm@latest`. This command globally installs the newest version of the npm client.

> 💡 **Tip:** The `-g` flag ensures npm is installed globally, making it available across all your projects.

### Step 6: Install Your First npm Package

To confirm that npm is fully functional, try installing a simple package. For example, you can install a global package like 'nodemon' by running `npm install -g nodemon` in your terminal. This will download and install the specified package, demonstrating npm's capability to manage dependencies.

> 💡 **Tip:** If you're new to the command line, check out our [How to Install npm: A Beginner's Step-by-Step Guide](/en/how-to-install-npm-beginner-guide) for more detailed instructions and troubleshooting for common issues.


## Tips & Tricks

- Consider using a Node Version Manager (NVM) like nvm for macOS/Linux or nvm-windows for Windows if you need to switch between different Node.js and npm versions for various projects.
- Understand the difference between global (`npm install -g`) and local (`npm install`) package installations. Global packages are available system-wide, while local packages are specific to a project's directory.
- On Windows, always run your Command Prompt or PowerShell as an administrator when performing global npm installations to avoid permission errors.

## Warnings

- ⚠️ Always download Node.js installers from the official nodejs.org website to prevent installing malicious software.
- ⚠️ Be cautious when using `sudo` (superuser do) commands on Linux or macOS for global installations, and ensure you understand what the command does before executing it.

## Frequently Asked Questions

**How do I install npm?**

The most common and recommended way to install npm is by installing Node.js, as npm is automatically bundled with it. Visit nodejs.org, download the appropriate installer for your operating system, and follow the installation wizard.

**How to install npm in Windows?**

To install npm in Windows, download the Windows Installer (.msi) from the official Node.js website (nodejs.org). Run the installer and proceed with the default options; it will automatically install both Node.js and npm on your system.

**How to install npm on Mac?**

On a Mac, you can install npm by downloading the macOS Installer (.pkg) from nodejs.org and running it. Alternatively, many developers use Homebrew, a package manager, by running `brew install node` in their terminal.

**How to install npm in Ubuntu?**

To install npm in Ubuntu, you typically install Node.js first. You can use your system's package manager with `sudo apt update && sudo apt install nodejs npm` for older versions, or add the NodeSource repository for the latest stable releases.

**Can I install npm without Node.js?**

While it's technically possible to install npm separately, it is not recommended for most users. npm is tightly integrated with Node.js, and installing Node.js is the most straightforward, stable, and supported method to get npm working.

**How do I check if npm is installed on my system?**

To check if npm is installed, open your terminal or command prompt and type `npm -v`. If npm is correctly installed, it will display its version number. If it's not installed, you will likely receive a 'command not found' error message.
