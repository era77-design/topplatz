npm (Node Package Manager) is an essential tool for JavaScript developers, providing access to a vast ecosystem of open-source packages. It comes bundled with Node.js, simplifying the installation process. This guide will walk you through how to install npm, ensuring you're ready to manage project dependencies efficiently.

## What You Need

- Internet connection
- Administrator or root privileges

## Steps

### Step 1: Check for Existing Node.js and npm Installation

Before proceeding, it's wise to check if Node.js and npm are already installed on your system. Open your terminal (macOS/Linux) or Command Prompt/PowerShell (Windows) and type `node -v` and then `npm -v`. If versions are returned, they are installed. If not, or if you want to update, continue with the next steps.

> 💡 **Tip:** If you see version numbers, you might only need to update npm, which can be done with `npm install -g npm@latest`.

### Step 2: Download the Node.js Installer

The most straightforward way to install npm is by installing Node.js, as npm is included with it. Visit the official Node.js website (nodejs.org) and download the recommended LTS (Long Term Support) version for your operating system. This version is stable and suitable for most users.

> 💡 **Tip:** Always download installers from official sources to ensure authenticity and security. Avoid third-party sites for core development tools.

### Step 3: Run the Node.js Installer

Once the installer file (e.g., .msi for Windows, .pkg for macOS) is downloaded, locate it and double-click to start the installation wizard. Follow the on-screen prompts, accepting the license agreement and choosing the default installation location and components. Ensure that 'npm package manager' is selected for installation.

> 💡 **Tip:** For most users, the default installation options are perfectly fine and recommended. You typically don't need to customize the components unless you have specific requirements.

### Step 4: Complete the Installation Process

The installer will guide you through the remaining steps. This usually involves clicking 'Next' a few times and then 'Install'. You may be prompted for administrator credentials during this phase. Once the installation is complete, click 'Finish' to exit the wizard.

> 💡 **Tip:** After installation, it's often a good idea to restart your terminal or command prompt to ensure that the system's PATH variable is correctly updated and recognizes the new Node.js and npm commands.

### Step 5: Verify the Installation

After the installation is complete, open a *new* terminal or command prompt window. Type `node -v` and press Enter, then type `npm -v` and press Enter. You should see the version numbers for both Node.js and npm displayed, confirming a successful installation. If you encounter issues, double-check your installation steps.

> 💡 **Tip:** If commands are not recognized, try restarting your computer. This can sometimes resolve PATH issues, especially on Windows.

### Step 6: Update npm to the Latest Version (Optional but Recommended)

While npm comes bundled with Node.js, it might not always be the absolute latest version. To ensure you have the most up-to-date features and security patches, you can update npm separately. In your terminal, run the command: `npm install -g npm@latest`. This command globally installs the newest version of npm.

> 💡 **Tip:** The `-g` flag ensures npm is installed globally, making it available for all your projects. Regularly updating npm helps prevent compatibility issues with newer packages.


## Tips & Tricks

- Consider using a Node Version Manager (NVM) like `nvm` for macOS/Linux or `nvm-windows` for Windows. NVM allows you to easily install and switch between different Node.js and npm versions, which is highly beneficial for managing multiple projects with varying requirements.
- For Linux users, installing Node.js via your distribution's package manager (e.g., `apt` for Ubuntu, `yum` for CentOS) is an alternative. However, these versions can sometimes be older than the official Node.js website's releases. For the latest features, the official installer or NVM is often preferred.
- If you encounter permission errors during global package installations, especially on macOS or Linux, avoid using `sudo` with `npm`. Instead, configure npm to use a different directory or use NVM, which handles permissions more gracefully. For more detailed guidance, refer to [How to Install npm: A Complete Step-by-Step Guide](/en/how-to-install-npm).

## Warnings

- ⚠️ Always download Node.js installers from the official nodejs.org website to prevent installing malicious or outdated software. Using unofficial sources can compromise your system's security.
- ⚠️ Ensure you have administrative rights on your computer before attempting to install Node.js and npm, as the installation process modifies system files and directories.

## Frequently Asked Questions

**How to install npm in Windows?**

To install npm in Windows, the simplest method is to download the official Node.js installer (.msi) from nodejs.org. Run the installer and follow the prompts, ensuring the 'npm package manager' component is selected. npm will be installed automatically alongside Node.js.

**How to install npm on Mac?**

Installing npm on Mac is best done by downloading the Node.js installer (.pkg) from the official Node.js website. Execute the installer and proceed through the wizard. Alternatively, you can use a package manager like Homebrew (`brew install node`) or a Node Version Manager (NVM) for more flexibility.

**How to install npm on Ubuntu?**

To install npm on Ubuntu, you can either use the official Node.js installer, a Node Version Manager (like `nvm`), or your system's package manager. For the latest versions, using NVM or the official installer is recommended. For a general system-wide installation, you might use `sudo apt update && sudo apt install nodejs npm`.

**What is npm and why do I need it?**

npm (Node Package Manager) is the default package manager for Node.js. It allows developers to share and reuse code, manage project dependencies, and automate development tasks. You need it to install libraries, frameworks, and tools necessary for modern JavaScript development.

**Can I install npm without Node.js?**

No, npm is bundled with Node.js and cannot be installed separately or function independently. Node.js provides the JavaScript runtime environment that npm uses to execute its commands and manage packages. For a comprehensive overview, see [How to Install npm: A Step-by-Step Guide for All OS](/en/how-to-install-npm-guide).

**How do I check if npm is installed correctly?**

To verify if npm is installed correctly, open a new terminal or command prompt and type `npm -v`. If you see a version number displayed, npm is successfully installed and recognized by your system. You can also check Node.js with `node -v`.

**What should I do if the npm installation fails?**

If your npm installation fails, first ensure you have administrative privileges. Try restarting your computer and running the installer again. Check for any error messages during the installation process, which can provide clues. You might also consider using a Node Version Manager (NVM) as an alternative installation method.
