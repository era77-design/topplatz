npm, the Node.js package manager, is essential for JavaScript development, allowing you to install and manage project dependencies. It comes bundled with Node.js, meaning installing Node.js will automatically install npm on your system. For a comprehensive overview, refer to our [How to Install npm: A Step-by-Step Guide for Beginners](/en/how-to-install-npm-guide).

## What You Need

- Computer (Windows, macOS, or Linux)
- Internet connection
- Administrator privileges

## Steps

### Step 1: Check for Existing Installation

Before proceeding, it's wise to check if Node.js and npm are already present on your system. Open your terminal or command prompt and run `node -v` and `npm -v`. If you see version numbers displayed, you already have them installed; otherwise, you're ready to proceed with a fresh installation.

### Step 2: Download Node.js Installer

Visit the official Node.js website at nodejs.org. Download the recommended LTS (Long Term Support) version for your specific operating system (Windows, macOS, or Linux). The LTS version is generally more stable and suitable for most users, ensuring compatibility and reliability.

### Step 3: Run the Node.js Installer

Execute the installer file you just downloaded. Follow the on-screen prompts carefully, accepting the license agreement and choosing the default installation options. It's crucial to ensure that the 'npm package manager' component is selected for installation, as it's typically bundled with Node.js by default.

### Step 4: Complete Installation

The installer will now proceed to copy all necessary files to your system. Once the installation process is complete, click 'Finish' or 'Close' to exit the setup wizard. While not always strictly necessary, opening a new terminal window or restarting your computer can help ensure that the system's PATH variables are correctly updated.

### Step 5: Verify npm Installation

Open a new terminal or command prompt window after the installation. This step is important because it ensures your system loads the updated PATH variables. Type `node -v` and `npm -v` again; you should now see the installed version numbers for both Node.js and npm, confirming a successful setup.

### Step 6: Update npm (Optional but Recommended)

It's good practice to ensure your npm installation itself is up to date, as the bundled version might not be the absolute latest. In your terminal, run the command `npm install -g npm@latest`. This command globally updates npm to its newest stable version, providing access to the latest features and security fixes.

### Step 7: Test npm by Installing a Package

To confirm that npm is fully functional and ready for use, try installing a simple package globally. For example, execute `npm install -g cowsay` in your terminal. If the installation completes without any errors and you can run the `cowsay` command, your npm setup is correctly configured and ready for your projects.


## Tips & Tricks

- **Use a Node Version Manager (NVM)**: For developers who work with multiple projects requiring different Node.js versions, consider using NVM (Node Version Manager). It allows you to easily switch between Node.js and npm versions. For more advanced installation methods, see our [How to Install npm: Step-by-Step Guide for Developers](/en/how-to-install-npm-step-by-step-guide).
- **Keep npm Updated**: Regularly update npm using `npm install -g npm@latest` to benefit from the latest features, bug fixes, and security patches. This helps maintain a secure and efficient development environment.
- **Understand Global vs. Local Installs**: `npm install -g <package>` installs a package globally (available system-wide), while `npm install <package>` installs it locally to your project's `node_modules` folder. Choose wisely based on whether the tool is for system-wide use or specific to a project.

## Warnings

- ⚠️ **Administrator Privileges**: Always ensure you have administrator or root privileges when running the Node.js installer or performing global npm updates/installations. Without proper permissions, the installation may fail or lead to permission errors when attempting to use npm.

## Frequently Asked Questions

**How do I install npm on Windows?**

To install npm on Windows, download the Node.js .msi installer from nodejs.org, which bundles npm automatically. Run the installer and follow the default prompts to complete the installation smoothly.

**What's the process to install npm on macOS?**

On macOS, you can install npm by downloading the Node.js .pkg installer from nodejs.org and running it like any other application. Alternatively, power users often use Homebrew (`brew install node`) which also includes npm by default.

**How can I install npm on Ubuntu or other Linux distributions?**

For Ubuntu and most other Linux distributions, the recommended way to install npm is through your system's package manager, typically `sudo apt install nodejs npm`. This ensures you get a stable version integrated with your system.

**What is npm and why do I need it?**

npm (Node Package Manager) is the default package manager for Node.js. You need it to install, manage, and share JavaScript packages, libraries, and tools for your Node.js projects, making development more efficient.

**How do I check if npm is already installed?**

To check if npm is installed, open your terminal or command prompt and type `npm -v`. If it's installed, its version number will be displayed; otherwise, you'll receive a 'command not found' error.

**Can I install npm without Node.js?**

No, npm is intrinsically bundled with Node.js. When you install Node.js, npm is automatically installed alongside it as its primary package manager. You cannot install npm as a standalone tool.

**What is the recommended way to install Node.js and npm?**

The most straightforward and recommended method for beginners is to download the official Node.js installer (LTS version) directly from nodejs.org. This ensures you get a stable Node.js version with a compatible npm.
