npm, or Node Package Manager, is a command-line tool that comes with Node.js and is used for managing packages and dependencies in JavaScript projects. Installing npm is straightforward, as it's typically bundled with the Node.js installer, making it accessible across various operating systems like Windows, macOS, and Linux.

## What You Need

- Computer (Windows, macOS, or Linux)
- Internet connection
- Command-line interface (Terminal, PowerShell, Command Prompt)

## Steps

### Step 1: Understand npm and its Prerequisites

Before you begin, it's crucial to understand that npm is the default package manager for Node.js. This means that when you install Node.js, npm is automatically installed alongside it. Node.js is an open-source, cross-platform JavaScript runtime environment that executes JavaScript code outside a web browser, and npm helps manage the modules (packages) for Node.js projects.

### Step 2: Check for Existing Node.js and npm Installations

Open your computer's command-line interface (Terminal on macOS/Linux, Command Prompt or PowerShell on Windows). Type `node -v` and press Enter, then type `npm -v` and press Enter. If Node.js and npm are already installed, these commands will display their respective version numbers. If you receive an error message like 'command not found,' then you'll need to proceed with the installation.

> 💡 **Tip:** If versions are displayed but are very old, consider updating them or performing a fresh installation.

### Step 3: Download the Node.js Installer

Navigate to the official Node.js website (nodejs.org) in your web browser. You will typically see two recommended versions for download: the LTS (Long Term Support) version and the Current version. For most users, especially beginners, the LTS version is highly recommended as it offers greater stability and reliability for production environments. Download the installer appropriate for your operating system (Windows, macOS, or Linux).

> 💡 **Tip:** Always download from the official Node.js website to ensure security and authenticity.

### Step 4: Run the Node.js Installation Wizard

Locate the downloaded installer file (e.g., `.msi` for Windows, `.pkg` for macOS) and double-click to run it. Follow the prompts in the installation wizard: accept the license agreement, choose the installation directory, and ensure that 'npm package manager' is selected for installation (it usually is by default). Complete the installation by clicking 'Install' and then 'Finish'.

> 💡 **Tip:** It's generally safe to accept the default installation settings unless you have specific requirements.

### Step 5: Verify the Installation

After the installation is complete, it's essential to verify that both Node.js and npm have been successfully installed and are accessible from your command line. Close your current terminal or command prompt window and open a *new* one. Run `node -v` and `npm -v` again. You should now see the version numbers displayed, confirming a successful installation. For a more detailed walkthrough specifically aimed at new users, you can also refer to our [How to Install npm: A Beginner's Step-by-Step Guide](/en/how-to-install-npm-beginner-guide).

> 💡 **Tip:** Always open a new terminal after installation to ensure that the system's PATH variable is correctly updated.

### Step 6: Update npm (Optional but Recommended)

While npm is installed with Node.js, it often receives updates more frequently than Node.js itself. To ensure you have the latest version of npm, which includes the newest features and security patches, run the command `npm install -g npm@latest` in your terminal. The `-g` flag ensures that npm is updated globally on your system. This step is optional but highly recommended for optimal performance and security.

> 💡 **Tip:** Regularly updating npm can prevent compatibility issues with newer packages.

### Step 7: Start Using npm

With npm successfully installed and verified, you are now ready to start managing packages for your JavaScript projects. You can initialize a new project with `npm init`, install dependencies with `npm install <package-name>`, or run scripts defined in your `package.json` file. Explore the vast npm registry to find packages that can enhance your development workflow and project functionalities.

> 💡 **Tip:** Familiarize yourself with basic npm commands to streamline your development process.


## Tips & Tricks

- Always use the LTS (Long Term Support) version of Node.js for production environments due to its stability and long-term maintenance.
- Restart your terminal or command prompt after installing Node.js and npm to ensure that the system's PATH environment variable is correctly updated.
- Regularly update npm using `npm install -g npm@latest` to access the newest features, performance improvements, and security patches.

## Warnings

- ⚠️ Ensure you download the Node.js installer exclusively from the official Node.js website (nodejs.org) to avoid potential security risks from unofficial sources.
- ⚠️ Be cautious when globally installing packages (`npm install -g`). While convenient, global installations can sometimes lead to version conflicts between different projects. Consider using a Node Version Manager (like nvm) if you frequently work on projects requiring different Node.js or npm versions.

## Frequently Asked Questions

**How do I install npm?**

You install npm by installing Node.js. npm is bundled with the Node.js installer, so once Node.js is successfully installed on your system, npm will be available for use. Simply download the Node.js installer from nodejs.org and follow the installation wizard.

**How to install npm in Windows?**

To install npm in Windows, download the `.msi` installer for Node.js from the official Node.js website. Run the installer and proceed through the setup wizard, accepting the default options. Node.js and npm will be automatically added to your system's PATH, making them accessible from Command Prompt or PowerShell.

**How to install npm on Mac?**

On macOS, you can install npm by downloading the `.pkg` installer from nodejs.org and following the guided installation steps. Alternatively, if you use Homebrew, you can install Node.js (and thus npm) by running `brew install node` in your terminal.

**How to install npm on Ubuntu?**

For Ubuntu and other Debian-based Linux distributions, the recommended way to install npm (and Node.js) is by using the NodeSource PPA for the latest versions. You can also use `sudo apt install nodejs npm` to install the versions available in your system's default repositories, though these might be older.

**What if `npm -v` doesn't work after installation?**

If `npm -v` doesn't work, first ensure you've opened a *new* terminal or command prompt window after installation. If the issue persists, verify that Node.js and npm were added to your system's PATH environment variables. Sometimes, re-running the installer or manually adding the paths can resolve the problem.

**Do I need to install npm separately from Node.js?**

No, you do not need to install npm separately. npm is automatically included and installed when you download and install Node.js. The Node.js installer ensures that both the runtime environment and its package manager are set up on your system simultaneously.
